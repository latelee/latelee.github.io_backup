# 程序设计原则
## C++
C++11特性只选用合适的，模板排查问题时不好排查。
匿名函数，如代码行数少，可用，如多，单独一个函数。
命名空间不要太过如：
boost::asio::ip::tcp
std::chrono::time_point<std::chrono::high_resolution_clock> begintime;

少用try-catch。


简单的atoi用C风格的。

如果复杂的类型，用auto，简单的，不用。

stl：用list queue vector array map等。
